# KaminskyAICoreBot

Telegram bot for translation document processing service built with Zig.

## Architecture
- **Language**: Zig 0.14 (compiled via Docker on Railway)
- **HTTP**: httpz (karlseguin/http.zig) - cloned as path dependency
- **Database**: SQLite with WAL mode (on Railway Volume at /data/db/bot.db)
- **File Storage**: Railway Volume at /data/files/
- **Frontend**: Telegram Mini App (vanilla JS, served via @embedFile)

## Key Integrations
- Telegram Bot API (webhook mode)
- OpenAI GPT-4.1 Nano (glossary extraction via Batch API)
- DeepL API v3 (glossary management + translation)
- Stripe (invoicing via Checkout Sessions)

## Project Structure
- `src/` - Zig source code
- `web/` - Mini App frontend (HTML/CSS/JS)
- `sql/` - SQLite migrations
- `libs/` - Dependencies (httpz, sqlite3.c) - created during Docker build

## Environment Variables
See `.env.example` for required configuration.

## Deployment
Railway with Docker builder. Volume mounted at `/data`.
